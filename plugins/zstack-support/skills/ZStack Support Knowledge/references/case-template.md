# 案例模板

## 目录命名

```text
cases/YYYYMMDD-客户或项目主题/
```

一个独立的支持问题等于一个案例目录。拆分独立问题到单独的案例目录，在 notes.md 中交叉引用。

## 推荐结构

```text
case-directory/
  raw/          # 仅本地，不提交
  extracted/    # 仅本地，不提交
  outputs/      # 仅本地，除非脱敏并获批准否则不提交
  notes.md
  final_report.md
```

## 紧凑元数据

```markdown
- 案例 ID：
- 案例目录：
- 问题边界：
- 关联案例目录：
- 来源：
- 问题域：
- 当前证据状态：已确认 / 较可能 / 可能 / 证据缺失
- 证据级别：E0 / E1 / E2 / E3 / E4 / E5
- 闭环状态：需要更多证据 / 需要客户验证 / 临时规避 / 追踪已知缺陷 / 新缺陷草案 / 非产品缺陷 / 质量治理 / 知识更新 / 分析修订 / 需要研发决策 / 已关闭
- 下一步行动：
- 下一责任人：
```

## 证据条目格式

```markdown
### 证据条目

- 来源：
- 产物或位置：
- 时间戳：
- 关键摘录摘要：
- 公开标签：已确认 / 较可能 / 可能 / 证据缺失
- 证据级别：E0 / E1 / E2 / E3 / E4 / E5
- 边界：当前证据 / 源码参考 / 手册参考 / ZStack知识社区(BBS) 参考 / 公开 Web 参考 / Jira 内部参考 / Confluence 内部参考
```

## 参考条目格式

```markdown
### 参考条目

- 参考类型：GitHub 源码 / 手册 / ZStack知识社区(BBS) / 公开 Web / Jira / Confluence
- 来源状态：工具可见 / 已连接 / 结构化查询成功 / MCP 查询未完成 / 未配置
- 仓库、手册或页面：
- Ref、版本或页面日期：
- 路径或章节：
- 查询词：
- 机制或相似性摘要：
- 证据边界：
- 当前案例仍需验证：
```

Jira/Confluence 参考条目必须脱敏，禁止记录内部 URL、账号、Token、原始页面内容、原始工单描述、评论原文或未脱敏附件。
